# ALCHEM Lab Code Repository

본 GitHub 저장소는 **ALCHEM Lab 내부에서 사용하는 계산 및 분석 코드를 체계적으로 관리하고 공유하기 위해** 만들어졌습니다.

각 구성원은 자신이 작성한 코드를 용도에 맞는 폴더에 정리해 주세요. 새로운 코드를 추가할 때는 코드의 사용 방법과 실행 환경을 설명하는 `README.md` 파일을 반드시 함께 작성해야 합니다.

## 폴더 구성

### `DFT_calc_set`

DFT 계산 전후의 작업을 지원하는 코드를 정리하는 폴더입니다.

`INCAR`, `POSCAR`, `POTCAR`, `OUTCAR` 등 VASP 입출력 파일과 관련된 코드가 파일 종류와 용도에 따라 각 하위 폴더에 정리되어 있습니다.

### `Bader`

DFT 계산 결과를 이용한 **Bader charge 분석** 관련 코드를 정리하는 폴더입니다.

### `lobster`

DFT 계산 결과를 이용한 **COHP 분석** 관련 코드를 정리하는 폴더입니다.

### `DOS_plot`

DFT 계산 결과의 **Density of States(DOS) 분석 및 시각화** 관련 코드를 정리하는 폴더입니다.

### `Bandstructure`

DFT 계산 결과의 **Band structure 분석 및 시각화** 관련 코드를 정리하는 폴더입니다.

## 코드 관리 규칙

1. 코드의 수정은 원칙적으로 해당 코드의 작성자만 진행합니다.
2. 코드를 업로드할 때는 같은 폴더에 `README.md`를 작성하여 다음 내용을 안내해 주세요.
   - 코드의 목적과 주요 기능
   - 필요한 실행 환경과 라이브러리
   - 입력 파일 및 출력 파일
   - 실행 방법과 사용 예시
   - 코드 작성자 및 문의 방법
3. 코드에서 오류나 개선이 필요한 부분을 발견하면 직접 수정하지 말고 GitHub `Issues`에 내용을 등록해 주세요.
4. 등록된 이슈는 해당 코드의 작성자가 확인하여 수정한 후 저장소에 반영합니다.
5. 다른 작성자의 코드를 수정해야 한다면 반드시 작성자와 먼저 상의해 주세요.

## Git 사용 시 주의사항

작업을 시작하기 전에 반드시 원격 저장소의 최신 변경 사항을 먼저 반영해 주세요.

```bash
git pull origin main
```

그다음 코드를 수정하거나 추가한 후 변경 사항을 커밋하고 업로드합니다.

```bash
git add .
git commit -m "변경 사항을 설명하는 메시지"
git push origin <branch-name>
```

> [!CAUTION]
> `main` 브랜치와의 병합은 충돌 및 코드 유실을 방지하기 위해 매우 신중하게 진행해 주세요.  
> 가능하면 별도의 브랜치에서 작업한 후 Pull Request를 통해 병합하고, 충돌이 발생하면 관련 코드 작성자와 상의하여 해결해 주세요.

모든 구성원이 코드를 안전하고 편리하게 활용할 수 있도록 위 규칙을 준수해 주시기 바랍니다.
